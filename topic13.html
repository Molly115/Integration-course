Event Driven Programming (EDP)

Building responsive applications that react to user actions and system notifications.

What is EDP?

A programming paradigm where the flow of the program is determined by events. Instead of following a strict top-to-bottom sequence, the code "waits" for something to happen.

  • User Actions: Clicks, key presses.
  • System Events: Sensor outputs, messages from other programs.
  • Efficiency: Perfect for GUIs and gaming software.

The Event Loop

The heart of EDP. It continuously monitors for events and dispatches them to the appropriate Event Handlers.

Python: Asyncio & Coroutines

In Python, we use the asyncio module to handle asynchronous tasks. Coroutines are specialized functions defined with async def.

import asyncio

async def main():
    print("Hello...")
    await asyncio.sleep(1) # Simulate a wait
    print("...World!")

asyncio.run(main())

Tasks (asyncio.Task)

Tasks are used to schedule coroutines concurrently. This allows the program to handle multiple asynchronous units of work at once without stopping the whole script.

Key Takeaways

  • Responsive: Apps stay active while waiting for data.
  • Scalable: Efficiently handles multiple tasks.
  • Modern: Essential for web servers and interactive desktop apps.
